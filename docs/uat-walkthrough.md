# DeepSleep 穿行测试 / UAT 操作手册

> **用途**：你一个人拿着本文件在浏览器里走完整条业务链，记录「哪里不顺、哪里怪、哪里缺」。  
> **版本**：v1.6.1 · 2026-06-04  
> **建议耗时**：45–60 分钟（含记录问题）  
> **环境**：`npm run dev` → 通常 `http://localhost:5173`

## 十、建议重点「挑刺」的 12 个点

走流程时特别留意这些（都是以往 brainstorm 里容易出问题的）：

1. 看板任务名（如 IT 风险评估）是否一眼像真项目还是像 demo 模板  
2. Scope 生成后任务数量是否过多/过少，有没有「不知道干嘛」的卡片  
3. 门禁页文案是否让人知道「下一步该干嘛」  
4. 看板拖拽失败时 Toast 是否看得懂  
5. 工作台左侧列表 `auditPhase` 是否还显示英文 key（如 `control-test`）  
6. 进度阶段条 6 行是否信息过载  
7. 双状态提示是否帮到你，还是仍然困惑  
8. 负责组筛选后 KPI 是否合理  
9. 成员邀请链接新开标签体验  
10. 项目列表搜索/排序是否够用  
11. 刷新后看板列 vs 工作台状态是否一致  
12. 整体中文文案是否统一（审计词 vs 研发词残留）

## 十一、快速冒烟（只有 15 分钟时）

最小路径：

```text
新建 Audit+ITA 项目（用 §零 数据）
→ 概览生成 Scope（银行 ITGC 模板）
→ 看板：IT 风险评估拖到已完成
→ 工作台：访问管理控制测试 填表+上传+待复核
→ 进度：看 KPI + 6 步阶段条 + 抽屉双状态
→ F5 刷新
```
